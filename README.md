# Open World Baltimore

Open World Baltimore is a Next.js + Cesium project for exploring Baltimore in 3D with photorealistic tiles and live event markers.

## Getting Started

Install dependencies and run locally:

```bash
npm ci
npm run dev
```

Open http://localhost:3000.

## Environment

Create `.env.local` with:

```env
CESIUM_ION_TOKEN=
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

Notes:
- `CESIUM_ION_TOKEN` is required for Cesium ion assets.
- Google Maps API key is required for Google photorealistic tiles.

## Event Markers

The `/play` scene now renders upcoming events as map markers.

- Event source: `https://codecollective.us/baltimore/upcoming_events.json`
- App endpoint: `GET /api/events`
- Server-side cache: in-memory TTL cache (15 minutes) with stale fallback
- Marker clustering is enabled for performance
- Clustered markers render as larger circles (no numeric text)

You can disable event markers for comparison runs with:

```text
/play?events=off
```

## Performance Testing (Automated)

This repo includes Playwright-based performance profiling that compares:

- `events_on`: `/play`
- `events_off`: `/play?events=off`

The test simulates interactions, captures Event Timing + Long Task metrics, and writes:

- `perf-results/inp-comparison.json`

### Run in Docker (recommended)

```bash
npm run perf:test:docker
```

Useful env vars:

```bash
PERF_WARMUP_RUNS=2 \
PERF_MEASURED_RUNS=10 \
PERF_FAIL_P95_MS=300 \
PERF_FILTER_OUTLIERS=1 \
npm run perf:test:docker
```

What this does:
- Adds event markers and clusters in the scene
- Profiles interaction performance with and without events
- Computes p50/p95/p99, mean, stdev, and 95% confidence intervals
- Applies a CI-style gate on `events_on` p95 (`PERF_FAIL_P95_MS`)

## Development Notes

- Generated profiling artifacts are gitignored (`perf-results/`, `test-results/`, `playwright-report/`).
- Cesium assets in `public/cesium` are regenerated by `postinstall`.
